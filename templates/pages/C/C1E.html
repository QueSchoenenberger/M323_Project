Lernnachweis zur Kompetenz C1E

Thema: Auswirkungsabschätzung und Gewährleistung der Code-Integrität nach Refactoring

Ausgangscode

def calculate_square_area(side_length):
    return side_length * side_length

def calculate_rectangle_area(length, width):
    return length * width

if __name__ == "__main__":
    # Example for the square area
    square_side_length = 5
    square_area = calculate_square_area(square_side_length)
    print(f"Area of a square with side length {square_side_length}: {square_area}")

    # Example for the rectangle area
    rectangle_length = 4
    rectangle_width = 6
    rectangle_area = calculate_rectangle_area(rectangle_length, rectangle_width)
    print(f"Area of a rectangle with length {rectangle_length} and width {rectangle_width}: {rectangle_area}")

Refactoring

def calculate_area(shape, *args):
    if shape == "square":
        return args[0] * args[0]
    elif shape == "rectangle":
        return args[0] * args[1]

if __name__ == "__main__":
    # Example for the square area
    square_side_length = 5
    square_area = calculate_area("square", square_side_length)
    print(f"Area of a square with side length {square_side_length}: {square_area}")

    # Example for the rectangle area
    rectangle_length = 4
    rectangle_width = 6
    rectangle_area = calculate_area("rectangle", rectangle_length, rectangle_width)
    print(f"Area of a rectangle with length {rectangle_length} and width {rectangle_width}: {rectangle_area}")

Auswirkungsabschätzung und Gewährleistung der Code-Integrität

  1. Auswirkungsabschätzung:
    • Durch das Refactoring wurde die Anzahl der Funktionen reduziert, um die Wartbarkeit zu verbessern und Redundanz zu vermeiden.
    • Die Funktionalität wurde in eine gemeinsame Funktion «calculate_area» zusammengefasst, um den Code zu vereinfachen.
  2. Tests:
    • Vor dem Refactoring wurden Tests für «calculate_square_area» und «calculate_rectangle_area» durchgeführt, um deren korrekte Funktionalität sicherzustellen.
    • Nach dem Refactoring müssen die Tests für «calculate_area» sicherstellen, dass alle Formen korrekt berechnet werden.
  3. Integrationstests:
    • Bestehende Integrationstests, die die Gesamtfunktionalität des Codes überprüfen, müssen aktualisiert werden, um das neue Refactoring zu berücksichtigen.
  4. Review:
    • Code-Reviews durch Teammitglieder sind wichtig, um sicherzustellen, dass das Refactoring verständlich und akzeptabel ist.
  5. Versionierung:
    • Sicherstellung, dass das Refactoring ordnungsgemäß in das Versionskontrollsystem integriert ist, um bei Bedarf zu früheren Versionen zurückkehren zu können.
  6. Monitoring:
    • Nach der Implementierung des Refactorings sollte der Code überwacht werden, um sicherzustellen, dass keine unerwarteten Nebeneffekte auftreten.

Die Schätzung der Auswirkungen und die Gewährleistung der Code-Integrität sind essenzielle Schritte, um die Qualität und Zuverlässigkeit des Codes nach einem Refactoring zu gewährleisten.
